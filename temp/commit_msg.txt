Phase 2.5 Run A: compile + inline LIF + I/O + tuning; ~30x throughput

Run A finished: 300M tokens on 5080 in 1.58h at ~53k tok/s (from ~2k initially).

Inference energy: spiking 20.5 nJ/tok flat in seq-len vs transformer
27-46 nJ/tok at seq 128-4096 = 1.32x to 2.23x spiking advantage,
growing with context length.

Optimizations in order of impact:
- Inline LIF cell in src/model.py compile_safe path: atan surrogate,
  vth=1, reset-by-subtraction. Replaces snntorch.Leaky which caused
  torch.compile graph breaks. Old snntorch path retained for Phase-1 tests.
- torch.compile mode=default wired into train_gpu.py with raw_model
  handle for SET and checkpoint access. Compiled wrapper's _orig_mod
  prefix would otherwise break state_dict compatibility.
- TF32 enabled via torch.set_float32_matmul_precision high on cuda.
- Sequential disk reads in src/data_stream.py: one bulk read + B
  sub-windows, ~10x less seeking on the 55GB train.bin memmap.
- Larger batch with grad_accum=1 at config level.

New: configs/phase25_b_tf.yaml -- nanoGPT-style transformer baseline at
19.89M params matched to Run B for the head-to-head comparison. NO BDH
baseline -- comparison is against a familiar standard transformer.
